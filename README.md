# Avaliação de Checkpoints 🚀

Bem-vindo ao repositório de entrega e avaliação dos seus Checkpoint! Este repositório utiliza uma automação com **Gemini** para analisar seu código e seus commits, agilizando o feedback do seu trabalho.

Siga atentamente as instruções abaixo para realizar a entrega do seu app.

## 📝 Instruções de Entrega

### 1. Fork do Repositório
Primeiro, crie uma cópia deste repositório para a sua conta do Github.
- Clique no botão **Fork** no canto superior direito desta página.
- Selecione sua conta pessoal como destino.

### 2. Clone o Repositório
Baixe o seu fork para a sua máquina local para começar a trabalhar.
```bash
git clone https://github.com/SEU_USUARIO/NOME_DO_REPOSITORIO.git
cd NOME_DO_REPOSITORIO
```

### 3. Desenvolvimento do App
Crie seu aplicativo dentro deste diretório. Sinta-se à vontade para usar as ferramentas e estruturas aprendidas em aula.

**Dicas importantes:**
- Mantenha a estrutura de pastas organizada.
- O código deve ser limpo e legível.

### 4. Commits
A avaliação também levará em conta a qualidade dos seus commits. Tente fazer commits pequenos e com mensagens descritivas.

Exemplos de boas mensagens:
- `feat: adiciona tela de login`
- `fix: corrige erro na validação do formulário`
- `docs: atualiza documentação do projeto`

Para commitar e enviar suas alterações:
```bash
git add .
git commit -m "feat: descrição do que foi feito"
git push origin main
```

### 5. Abrindo o Pull Request (PR)
Quando finalizar o desenvolvimento:
1. Vá até a página do seu repositório no GitHub.
2. Clique na aba **Pull Requests**.
3. Clique no botão **New Pull Request**.
4. Certifique-se de que a comparação é entre o seu fork (`head repository`) e este repositório original (`base repository`).
5. Dê o nome do PR baseado na sua turma, seguindo o modelo: Checkpoint 01 - <CURSO><TURMA>. Ex.: (Checkpoint 01 -  ADS C)
6. Coloque no corpo a Matrícula e o nome de cada integrante do grupo (Ex.? 160101999 - Adamastor Ostrogésio da Silva) e clique em **Create Pull Request**.

## 🤖 Avaliação Automática
Assim que você abrir o PR, um fluxo de trabalho (Workflow) será acionado. O **Gemini** analisará:
- A qualidade e estrutura do código do seu app.
- O histórico e as mensagens dos seus commits.

Fique atento aos comentários que o bot fará no seu PR!
